# Minesweeper3000

Minesweeper game with moving mines, leaderboards, and user accounts.

---

## How to run

**1. Create a virtual environment and install dependencies**

```
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

**2. Start the server**

```
fastapi dev
```

**3. Open the URL output in your browser**

The SQLite database (`minesweeper.db`) is created automatically on first run.

---

## How to play

### Login / SignUp
Enter a username and password. Your account is created automatically on signup.

### Starting a game
From the home screen, choose a difficulty

### Moving mines
Every 2 seconds, 10% of mines (chosen randomly at the start of each game) move to a random adjacent hidden, unflagged cell. Flagged mines do not move.

## Leaderboard

Tracks three things:
- **Fastest times** per difficulty (Easy, Medium, Hard)
- **Most wins** across all difficulties
- **Best win streak**

Custom games do not appear on the leaderboard.


### Tools

**FastAPI** handles HTTP routes and database manipulation.

**SQLModel** defines the database schema as Python classes. SQLite stores everything in a single file (`minesweeper.db`).

**Jinja2** renders HTML templates server-side, passing Python data into HTML.

**HTMX** requests fragments of HTML. Used in the game and navigation.

**JavaScript** renders the board from JSON returned by the server, sends reveal/flag actions back to the server and calls for the mvement of mines.
